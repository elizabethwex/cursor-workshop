Module Overview

Goal Confirm enrollment and understand how to get projects started using Cursor and GitHub.
Estimated time 5–10 min

Welcome to the Cursor Access Program

This program is designed to help designers work directly with code.

You do not need prior coding experience.

You do need curiosity.

You'll learn how to:

  • Navigate a shared codebase
  • Run a project on your local machine
  • Use Cursor as a design partner
  • Make a real contribution through GitHub
What This Program Is
  • This is not a tutorial in the traditional sense.
  • You will not be asked to memorize syntax.

Instead, you'll learn how to:

  • Move through unfamiliar systems safely
  • Ask good questions of code
  • Make small, confident changes
  • Understand how your work fits into a larger workflow
  • Everything here is reversible.
  • Nothing you do can permanently break the system.
How These Tools Work Together

You'll be working across three systems:

  • GitHub (Remote): This is where the shared version of the project lives online.
  • Your Computer (Local): This is your private working copy of the project.
  • Git: Git is the system that keeps these two in sync.

You’ll download a copy of a project to your computer, run it locally, make changes safely, and then propose updates back to GitHub. Nothing you do locally changes GitHub until you explicitly commit and push your work.

Prerequisites

Before working in Cursor, you will need the following:

  • Submit a ticket to access Cursor in Okta
  • Download the Cursor app
  • Node.js installed

To confirm Node.js is installed, you can run the command node -v in the terminal app (open on your mac). If you see a version number, you're good to go! If you don't see a version number, you can install it from the link above.

How Progress Works
  • The program is divided into modules.
  • Each completed module increases your access level and unlocks additional system capabilities.
  • Progress is saved automatically.
  • You may revisit any module at any time.
